# group4_vosir
#  Консольная игра ‘Числовая головоломка "15”’

Классическая головоломка, реализованная на языке Python. Игровое поле размером 4x4 содержит плитки с числами
от 1 до 15 и одну пустую клетку. Игрок должен перемещать плитки,
чтобы упорядочить их по возрастанию, перемещая их на свободное
место.
Особенности реализации: Двумерный массив для хранения
состояния игрового поля, функции для перемещения плиток и
проверки состояния игры.

##  Особенности проекта

- **Гарантия решаемости:** Алгоритм генерации проверяет количество инверсий, поэтому каждая новая игра гарантированно имеет решение.
- **Интерфейс:** Приятный консольный интерфейс с отрисовкой игрового поля.
- **Счетчик ходов:** Отслеживание эффективности прохождения.
- **Кроссплатформенность:** Работает на Windows, macOS и Linux (автоматическая очистка экрана).

##  Инструкция по запуску

Для запуска игры вам потребуется установленный интерпретатор **Python 3.x**. Дополнительные библиотеки устанавливать не нужно (используются только стандартные модули).

1. Скачайте код
    
    Убедитесь, что файл с игрой (например, game.py) находится в нужной директории.
    
2. Запустите игруBash
    
    Откройте терминал (командную строку) и выполните команду:
    
    python game.py
    

##  Управление

Для перемещения плиток используются стандартные клавиши WASD (регистр не важен):

W - Переместить плитку **ВВЕРХ** (на пустое место)

A - Переместить плитку **ВЛЕВО**

**S -** Переместить плитку **ВНИЗ**

D - Переместить плитку **ВПРАВО**

**Дополнительные команды:**

- **R** — Перезапустить игру (новая раскладка).
- **H** — Показать справку.
- **Q** — Выйти из игры.

##  Роли участников команды

Разработка велась командой из трех человек, каждый из которых отвечал за свой модуль:

###  Участник 1: Core Logic & Initialization

- **Зона ответственности:** Инициализация игры и математическая логика.
- **Реализовано:**
    - Функция  initialize_game:   создание поля и перемешивание чисел.
    - Функция is_solvable: алгоритм проверки решаемости комбинации (подсчет инверсий).
    - Базовая валидация позиций (is_valid_move).

###  Участник 2: UI & User Experience

- **Зона ответственности:** Визуализация и взаимодействие с пользователем.
- **Реализовано:**
    - Функция print_board: отрисовка таблицы и красивое форматирование вывода.
    - Функция show_help: меню справки и инструкций.
    - Очистка экрана под разные ОС.

###  Участник 3: Game Loop & Mechanics

- **Зона ответственности:** Игровой цикл и обработка движений.
- **Реализовано:**
    - Функция main: основной цикл игры (Game Loop).
    - Обработка ввода (get_direction) и перемещение плиток (move_tile).
    - Проверка условия победы (is_solved).
    - Логика доступных ходов (get_available_moves).
